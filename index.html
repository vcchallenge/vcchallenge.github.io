Introduction

Our workshop intends to gather technology that improves video compression beyond the DLVC with and without neural networks. Efficient video coding technology is the foundation and indispensable part of the image and signal processing field. Several series of video coding standards have been crafted in the past three decades, and several new ones are in the way coming. Techniques such as block partitioning, prediction mode and transform are becoming more sophisticated, and traditional video coding technology still shows strong vitality. With the development of deep learning and the proliferation of special hardware(e.g., GPUs), the method of recurrent neural network and convolutional neural network has been used to prove the potential of learning-based methods in the field of video compression.

Deep Learning-Based Video Coding (DLVC) features a set of deep learning-based coding techniques as well as new or improved non-learning-based coding techniques. To the best of our knowledge, this is the first open-source video coding software that integrates deep learning-based techniques into the traditional hybrid coding framework. It is quite promising: DLVC version 1.0 achieves more than 40% bits saving than H.265/HEVC at the same quality level! DLVC's outstanding performance and open source attributes make it an excellent platform for researching new video coding technologies.

This challenge is organized by the Future Video Coding Study Group (FVC-SG), which is affiliated with IEEE 1857 working group. We hope to attract researchers in the multimedia field, especially young students, to pay more attention to image and video compression. The winning program in the challenge will be recommended to the AVS working group of audio and video codec technical standards.

Challenge Tasks

The reference software for this challenge is DLVC, which includes both encoder and decoder functionality. The organizer welcomes any type of video compression technology. Participants are welcome to propose new technologies and integrate them into DLVC, as well as modify or remove any existing technologies in DLVC.

Submission Session: DLVC supports both Windows and Linux systems. If participants use the Windows system, the exe file needs to be submitted, otherwise the docker file needs to be submitted. In addition to this, Self-added files such as library files and trained models also need to be submitted simultaneously. All submitted files should be packaged and sent to vcchallenge@163.com.

Testing and Evaluation Session: The organizer will conduct a unified test of all tasks and submit the test results to the FVC meeting. The test set is provided by iQiyi. Overall BD Rate and Complexity will be the main indicators of evaluation. All evaluation and test results will be released on the Internet in time. It should be noted that participants should pay attention to the complexity of the technical proposal to ensure that the organizer can complete the test tasks within two weeks, otherwise it will be considered as giving up the challenge.

The awards process of this challenge will be synchronized with the FVC meeting held in Chengdu in June 2019. Everyone is welcome to participate actively. The specific meeting venue will be announced later.
